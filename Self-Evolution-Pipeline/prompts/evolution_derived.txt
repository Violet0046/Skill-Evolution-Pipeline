## Output format

Output a structured change description in this exact YAML-like format:

```
# Change 001
summary: <one-sentence description of the enhancement>

anchor:
  type: heading
  selector: "<exact heading text from skill>"

operation: INSERT_SUBSECTION

new_content: |
  <exact new content to add>
```

### Rules - CRITICAL

- **summary**: One sentence describing the enhancement
- **anchor.type**: Always "heading" (use the exact heading text as selector)
- **operation**: INSERT_SUBSECTION | INSERT_RULE | DELETE
- **new_content**: The exact new lines to add. Use | for multi-line content.
- Do NOT use code fences (no ``` blocks)
- Keep new_content minimal - just what you need to add

### Examples

**Example 1: Add a new subsection**
```
# Change 001
summary: Add output example section

anchor:
  type: heading
  selector: "### Step 1: 协议分析"

operation: INSERT_SUBSECTION

new_content: |
  ### 输出示例
  | 协议册 | 协议过程 | 信令信元 |
  |--------|----------|----------|
  | ...    | ...      | ...      |
```

**Example 2: Add a rule in existing section**
```
# Change 001
summary: Add input validation rule

anchor:
  type: heading
  selector: "## 输入"

operation: INSERT_RULE

new_content: |
  **输入约束**：需求文档必须存在于 input/ 目录。
```
