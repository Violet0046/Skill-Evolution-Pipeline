## 输出格式

以这种精确的 YAML 格式输出结构化变更描述：

```
# Change 001
summary: <增强内容的一句话描述>

anchor:
  type: heading
  selector: "<技能中的确切标题文本>"

operation: INSERT_SUBSECTION

new_content: |
  <要添加的确切新内容>
```

### 规则 - 关键

- **summary**: 描述增强的一句话
- **anchor.type**: 始终为 "heading"（使用确切标题文本作为选择器）
- **operation**: INSERT_SUBSECTION | INSERT_RULE | DELETE
- **new_content**: 要添加的确切新行。使用 | 表示多行内容。
- 不要使用代码块（不要 ``` 块）
- 保持 new_content 最小化——只添加你需要的

### 示例

**示例 1：添加新的小节**
```
# Change 001
summary: 添加输出示例章节

anchor:
  type: heading
  selector: "### Step 1: 协议分析"

operation: INSERT_SUBSECTION

new_content: |
  ### 输出示例
  | 协议册 | 协议过程 | 信令信元 |
  |--------|----------|----------|
  | ...    | ...      | ...      |
```

**示例 2：在现有章节中添加规则**
```
# Change 001
summary: 添加输入验证规则

anchor:
  type: heading
  selector: "## 输入"

operation: INSERT_RULE

new_content: |
  **输入约束**：需求文档必须存在于 input/ 目录。
```

## 自我评估

**如果增强清晰且可操作** — 在输出的最后一行包含 `{evolution_complete}`。

**如果你无法识别有价值的增强** — 仅输出：

{evolution_failed}
